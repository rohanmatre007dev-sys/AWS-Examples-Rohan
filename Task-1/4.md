# 4. Try different scenarios (high CPU usage, running multiple processes, network checks) and observe how the system responds.

### 1. High CPU usage scenario

### A. Create high CPU load

Run this in Terminal:

```bash
yes > /dev/null
```

This uses **100% of one CPU core**.

To use **multiple cores**:

```bash
yes > /dev/null &
yes > /dev/null &
yes > /dev/null &
yes > /dev/null &
```

(stop later using `killall yes`)

### B. Observe system response

Open another terminal and run:

```bash
top
```

or (better UI):

```bash
htop
```

üîç **What to observe**

- CPU % per core
- Load average
- System responsiveness (mouse lag, app delay)
- Fan speed / heat (if on laptop)
