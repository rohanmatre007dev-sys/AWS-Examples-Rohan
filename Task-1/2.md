### PART 1: FILE & DIRECTORY COMMANDS (Basics ‚Üí Practical)

### Step 1: Know where you are

```bash

```

**What it does:**

![image.png](attachment:795f2ba5-5b16-4953-9316-3420cced252c:image.png)

## Step 3: Move between directories

```bash
cd /etc
cd ..
cd ~
```

| Command | Meaning |
| --- | --- |
| `cd /path` | Go to directory |
| `cd ..` | Go one level up |
| `cd ~` | Go to home directory |

**IMP Points to remember:**

![image.png](attachment:9c2c7937-5df3-4c07-b5b2-789fb24b6016:image.png)

`/etc` stores **system-wide configuration files** that control how Ubuntu and its services behave.

**Mental Model (Very Important):**

| Directory | Purpose |
| --- | --- |
| `/etc` | Configuration |
| `/bin` | Essential commands |
| `/usr/bin` | User programs |
| `/var` | Logs & changing data |
| `/home` | User files |
|  |  |

## üü• MEMORY DETAILS (VERY COMMON CONFUSION)

Example:

```
VIRT: 48.9G
RES: 384M
```

### What this REALLY means:

- **VIRT (virtual)** = memory *address space* (can be misleading)
- **RES (resident)** = real RAM actually used

üëâ **Always trust RES**, not VIRT.
