## What is Git?

**Git** is a **free & open-source distributed version control system** used to track code changes, collaborate with teams, and manage versions efficiently.

* Created by **Linus Torvalds** (creator of Linux)

## What is Version Control?

Version control helps:

* Track who changed what
* Maintain complete code history
* Roll back to previous versions
* Collaborate safely with teams

---

## Centralized vs Distributed VCS

| Centralized (SVN)       | Distributed (Git)          |
| ----------------------- | -------------------------- |
| Single server           | Every user has full copy   |
| Single point of failure | No single point of failure |
| Needs network           | Works offline              |

## Git vs GitHub

* **Git** ‚Üí Version control tool (local)
* **GitHub** ‚Üí Platform for hosting Git repositories + collaboration + CI/CD

üëâ *Git = engine, GitHub = service built on top of Git*

## What is the `.git` Folder?

* Hidden folder created by `git init`
* Stores commit history, objects, branches, and configuration
* If deleted ‚Üí repository tracking is lost

## Git Workflow (Most Important)

```
git add ‚Üí git commit ‚Üí git push
```

* `git add` ‚Üí Stage changes
* `git commit` ‚Üí Save snapshot
* `git push` ‚Üí Send changes to remote repository

## `.gitignore`

### Why use `.gitignore`?

* Hide sensitive data (API keys, secrets)
* Ignore system files (.DS_Store, Thumbs.db)
* Ignore generated files (logs, builds)
* Ignore dependencies (node_modules, venv)

### Example `.gitignore`

```
.DS_Store
secrets.txt
node_modules/
*.log
```

## Git Squash

### 1. Interactive Rebase

```
git rebase -i HEAD~N
```

Change `pick` ‚Üí `squash` to combine commits.

If already pushed:

```
git push --force-with-lease
```

### 2. Squash Merge

```
git checkout main
git merge --squash feature-branch
git commit -m "Feature complete"
```
